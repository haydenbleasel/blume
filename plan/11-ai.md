# AI

## Goals

AI should make docs easier to consume and maintain without making the core docs site dynamic by default.

Blume should support:

- `llms.txt`
- `llms-full.txt`
- copy page as Markdown
- Ask AI over docs
- generated summaries
- generated metadata suggestions
- future MCP hooks for docs-aware agents

## Principles

- AI is optional.
- Static docs stay static when AI is disabled.
- Runtime AI features require server output.
- Default model calls use the AI SDK with Vercel AI Gateway model strings.
- Do not require provider-specific SDK packages for the default path.
- Fail fast with a clear server-side error when gateway auth is missing.

## LLM files

Generate:

```txt
/llms.txt
/llms-full.txt
```

Inputs:

- content graph
- page summaries
- route hierarchy
- API references
- canonical URLs

Config:

```ts
ai: {
  llmsTxt: true,
}
```

## Ask AI

Config:

```ts
ai: {
  ask: {
    enabled: true,
    model: "openai/gpt-4.1-mini",
  },
}
```

Runtime shape:

- React island for the chat UI
- Astro endpoint or action for the server call
- AI SDK for generation/streaming
- Vercel AI Gateway model string
- retrieval from local/generated docs index

If `ai.ask.enabled` is true and `deployment.output` is `static`, build should explain that Ask AI needs server output.

## Retrieval

Initial options:

- lexical search over built docs records
- Pagefind-derived index
- generated compact Markdown corpus

Later options:

- embeddings
- vector store integration
- Vercel Blob for generated artifacts
- Vercel Postgres or other storage integrations

## Generated docs helpers

Potential commands:

```bash
blume ai summarize docs/api
blume ai frontmatter docs/**/*.mdx
blume ai broken-links
```

These should be explicit commands that produce diffs or suggestions, not silent mutations.

## MCP

Implemented (`mcp.enabled`, server output): a hosted Streamable-HTTP MCP server
at `mcp.route` (default `/mcp`) with `.well-known` discovery, exposing read-only
tools:

- search docs (`search_docs`)
- fetch page Markdown (`get_page`)
- list routes (`list_pages`)
- navigation tree (`get_navigation`)

Future additions:

- inspect API references (once OpenAPI is parsed to a native IR)
- suggest edits

This makes Blume docs useful to coding agents without scraping the built site.

## Security and privacy

AI features must document:

- what content is sent to models
- how auth is configured
- how caching works
- whether user queries are stored
- how to disable telemetry

No AI feature should silently send private docs to a third-party provider.
