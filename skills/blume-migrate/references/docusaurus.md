# Docusaurus → Blume

New coverage — no codemod ever existed for Docusaurus, so inventory carefully. Docusaurus is a React/Webpack site; navigation comes from `sidebars.js` (explicit or autogenerated) plus per-folder `_category_.json`. Admonitions are already directive syntax, which eases the content transform.

## Detect

- **`docusaurus.config.{js,ts}`** at the root.
- A **`sidebars.{js,ts}`** file and per-folder **`_category_.json`** files.
- Content under **`docs/`** (and often a `blog/` dir); `@docusaurus/*` deps.

## Config: `docusaurus.config` → `blume.config.ts`

Read `themeConfig`, `presets`, and `plugins`:

| Docusaurus | Blume |
| --- | --- |
| `title` | `title` |
| `tagline` | `description` |
| `themeConfig.navbar.title` / `.logo` | `title` / `logo` (move the image into `public/`) |
| `themeConfig.navbar.items` (doc/link items) | `navigation.tabs` (for section links) or a Header override (for external CTAs) |
| `themeConfig.colorMode.defaultMode` | `theme.mode` |
| `themeConfig.prism.theme` / `.darkTheme` | `markdown.codeBlocks.theme: { light, dark }` (map Prism theme names to Shiki themes, e.g. `github`/`github-dark`) |
| `url` + `baseUrl` | **`url` → drop** (`deployment.site` auto-detects: localhost in dev, the deploy URL on Vercel — don't hardcode it); `baseUrl` (when not `/`) → `deployment.base` route prefix |
| preset `docs.routeBasePath` | serve docs under that prefix via `content.sources` `prefix`, or set `content.root` |
| preset `docs.editUrl` / `themeConfig.editUrl` | `github` (owner/repo/branch) |
| `themeConfig.footer` | drop → Footer override |
| `themeConfig.announcementBar` | `banner` (`{ content, dismissible, id }`) |
| `i18n.locales` / `defaultLocale` | `i18n` |
| `themeConfig.algolia` | drop — Blume ships built-in search (Orama); remove the Algolia dep |

## Navigation: `sidebars.js` + `_category_.json`

- **Autogenerated sidebar** (`{ type: 'autogenerated', dirName: '...' }`) → Blume's default filesystem navigation. Just structure the folders; no config needed.
- **Explicit sidebar** (arrays of doc IDs, categories, links) → restructure into folders where possible; use `navigation.sidebar` only for shapes files can't express. A category `{ type: 'category', label, items }` → a folder (label → `meta.ts` `title`); `collapsed`/`collapsible` → `meta.ts` `display: "group"` + `collapsed`.
- **`_category_.json`** (`{ label, position, collapsed, link }`) → a folder `meta.ts`: `label`→`title`, `position`→`order`, `collapsed`→`collapsed` (with `display: "group"`), `link.type: 'generated-index'` → an `index` page in the folder.
- **Sidebar `position` frontmatter** (`sidebar_position`) → `sidebar.order`.

## Frontmatter

| Docusaurus | Blume |
| --- | --- |
| `title` / `description` | pass through |
| `id` | usually drop (routing is filesystem-based); use `slug` to pin a route |
| `slug` | `slug` |
| `sidebar_label` | `sidebar.label` |
| `sidebar_position` | `sidebar.order` |
| `sidebar_class_name` / `sidebar_custom_props` | drop (report) |
| `pagination_next` / `pagination_prev` | drop — pagination is automatic |
| `draft` | `draft` |
| `tags` | `search.tags` (or drop if only used for the blog tag index) |
| `keywords` | drop → use `seo` |
| `hide_table_of_contents` | drop (report — no per-page TOC toggle) |
| `image` | `seo.image` |

Remove any duplicated H1 in the body — `title` renders the H1.

## Components & content

- **Admonitions are already directives** — mostly passthrough: `:::note`, `:::tip`, `:::info`, `:::warning`, `:::danger` work as-is. Map **`:::caution`** → `:::warning` (or rely on Blume's `caution`→warning alias). Titled admonitions (`:::note[Title]`) work.
- **Tabs:** `<Tabs>`/`<TabItem label="…" value="…">` → `<Tabs>`/`<Tab title="…">`. Drop `groupId`/`value`; move `label` to `title`. Strip the `import Tabs from '@theme/Tabs'` lines.
- **`@theme/*` and `@site/*` imports** — Blume injects components globally, so strip `@theme/*` imports. Rewrite `@site/` asset/module aliases to relative paths (or `/public` URLs for assets).
- **`<Admonition>`** JSX (rare) → the matching `:::` directive.
- **`<details>`/`<summary>`** → `<Accordion>`/`<AccordionItem>` or leave as raw HTML (Blume renders it).
- **MDX version pitfalls** — Docusaurus v2 used **MDX v1** (looser); Blume uses modern MDX. Watch for: unescaped `<`/`{` in prose (must be escaped or fenced), HTML-style comments `<!-- -->` inside MDX (use `{/* */}`), and bare `style="..."` string attributes (must be `style={{…}}` objects). Fix these as build errors surface.

## Versioned docs

Docusaurus `versioned_docs/` + `versions.json` → **recommend migrating the latest version only** into `content.root`. If older versions must stay, put each under its own folder and wire a `navigation.selectors` entry of `kind: "version"` pointing at each version's path.

## Blog

A Docusaurus `blog/` plugin → Blume `type: blog` pages. Move posts into a folder, add `type: blog` + `date`/`authors` frontmatter; Blume generates the feed. `authors.yml` → inline author objects in each post's `authors` frontmatter.

## Icons

Docusaurus has no built-in icon-name system (icons are usually inline SVG/emoji or CSS). Where you add sidebar/tab icons, use Lucide names. Convert any FontAwesome/React-icons usage in content to Lucide `<Icon>` or drop it.

## Teardown

Remove `@docusaurus/*`, `@theme/*`, and Algolia deps; delete `docusaurus.config.*`, `sidebars.*`, `babel.config.js`, and the `src/` theme customizations (re-implement via `blume eject`/`theme.css` if needed). Repoint `start`/`build`/`serve` scripts → `blume dev`/`blume build`/`blume preview`; add `blume`.

## Dropped — report these

Custom React theme components and swizzled components (re-implement via `blume eject`), footer, Algolia config, `sidebar_custom_props`, `hide_table_of_contents`, client redirects plugin patterns that are dynamic (move to host rules), and any `@theme/*` component with no Blume equivalent.
