# Search

Blume ships local search with no hosted infrastructure. The default engine is
[Orama](https://orama.com), which indexes your content graph — so search works
in `blume dev` (with live updates) and `blume build` alike.

```ts blume.config.ts
search: {
  provider: "orama", // default
  indexing: { includeHiddenPages: false },
}
```

Press <Badge variant="accent">⌘K</Badge> (or `/`) to open search.

## How it works

<Steps>
  <Step title="Index">
    Blume builds a search index from your pages (titles, descriptions, and body
    text) and serves it at `/blume-search.json`.
  </Step>
  <Step title="Search">
    The header search modal loads the index and queries it in the browser with
    Orama — instant, client-side, and available in dev.
  </Step>
</Steps>

## Large sites: Pagefind

For very large docs, opt into [Pagefind](https://pagefind.app), which indexes
the built HTML and loads the index in shards on demand:

```ts blume.config.ts
search: {
  provider: "pagefind";
}
```

Pagefind indexes after `blume build`, so it is not available in `blume dev`.

## Excluding pages

Only content marked indexable is searched. Exclude a page with frontmatter:

```yaml
search:
  exclude: true
```

Hidden pages are excluded unless `search.indexing.includeHiddenPages` is set.
