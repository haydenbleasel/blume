# Theming

Blume's default theme is token-driven and works in light and dark mode without
any setup.

## Config tokens

```ts blume.config.ts
theme: {
  accent: "teal",   // a named preset or any CSS color
  radius: "md",     // none | sm | md | lg
  mode: "system",   // system | light | dark
}
```

## theme.css

Add a `theme.css` to your project to override any CSS variable. It layers after
Blume defaults and config tokens:

```css theme.css
:root {
  --blume-accent: oklch(0.68 0.14 180);
  --blume-radius: 0.5rem;
}

:root[data-theme="dark"] {
  --blume-background: oklch(0.16 0 0);
}
```

## Cascade order

<Steps>
  <Step title="Base">Blume reset and component styles.</Step>
  <Step title="Config">Tokens compiled from `theme` config.</Step>
  <Step title="theme.css">Your overrides win.</Step>
</Steps>

## Tailwind utilities

Blume's theme is built with Tailwind v4 internally, and your project's `.astro`
pages and components are scanned too — so you can use Tailwind utility classes
(including token-aware ones like `bg-accent` and `text-muted-foreground`)
without configuring Tailwind yourself.
