# Customization

Blume scales from zero config to full ownership. Override a component, add an
interactive island, mount a fully custom page, install source components, or
eject the whole runtime — reach for only the layer you need.

## Component overrides

Add a `components.ts` (or `components.tsx`) to your project root and export
`defineComponents`. The `mdx` map either **replaces** a built-in component or
**adds** a new one — available in every `.mdx` page with no import.

```ts components.ts lineNumbers
import { defineComponents } from "blume";
import Callout from "./components/Callout.astro";
import Pricing from "./components/Pricing.astro";

export default defineComponents({
  mdx: {
    Callout, // replace the built-in Callout
    Pricing, // add a new <Pricing /> component
  },
});
```

Keys are the names you write in MDX (`<Callout>`, `<Pricing>`). Use the `.tsx`
filename when you import React components.

## React islands

The core theme is React-free, but React switches on automatically the moment your
project contains a `.tsx` or `.jsx` file. To use an interactive component in MDX,
wrap it in an `.astro` file with a [hydration directive](https://docs.astro.build/en/guides/framework-components/#hydrating-interactive-components):

```astro Counter.astro lineNumbers
---
import Counter from "./Counter.tsx";
---

<Counter client:load />
```

Register the wrapper as an MDX override and use `<Counter />` in any page:

```ts components.ts lineNumbers
import { defineComponents } from "blume";
import Counter from "./Counter.astro";

export default defineComponents({
  mdx: { Counter },
});
```

Choose how eagerly the island hydrates with the directive: `client:load`,
`client:idle`, `client:visible`, `client:media`, or `client:only`.

## Custom pages

Add `.astro` files under your `pages/` folder to mount fully custom routes
alongside your docs. They keep their location, so relative imports and
`getStaticPaths` work as usual. Read your config, navigation, and page list from
the `blume:data` module:

```astro pages/changelog.astro lineNumbers
---
import data from "blume:data";
---

<h1>{data.config.title} changelog</h1>
```

## Registry

Install Blume-maintained components as **source** in your project — you own the
code and can edit it freely:

```bash
blume add feedback
```

The component is copied into your project rather than imported from the package,
so it's yours to change.

## Eject

When you want full control, eject the generated runtime into a standalone Astro
project:

```bash
blume eject --yes
```

Eject is a one-way step: the hidden `.blume/` runtime becomes a normal Astro app
you own and can modify directly. The `blume` package stays importable, so you
keep its components, theme, and Markdown processors.
