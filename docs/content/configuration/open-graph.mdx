# Open Graph images

Blume can generate a social share image for every page at build time — no
headless browser, no manual design work. Rendering uses
[Takumi](https://takumi.kane.tw), so there's no Chromium dependency and builds
stay fast.

## Enable

Open Graph images are off by default. Turn them on in `blume.config.ts`:

```ts blume.config.ts
og: {
  enabled: true,
}
```

Blume then renders a 1200×630 PNG for each route and wires up the meta tags.

## What the card looks like

Each card is derived from your content and theme — there's nothing to design:

- the **page title** as the headline
- your **site title** as the eyebrow
- your theme **accent** for the mark and underline
- a dark background tuned for legibility in social feeds

## Where images live

Images are served at `/og/<slug>.png`, mirroring each page's route:

| Page route          | Image URL                  |
| ------------------- | -------------------------- |
| `/`                 | `/og/index.png`            |
| `/quickstart`       | `/og/quickstart.png`       |
| `/configuration/ai` | `/og/configuration/ai.png` |

They're prerendered, so they ship as static files even in server mode.

## Absolute URLs

`og:image` needs an absolute URL to preview on most platforms. Set
`deployment.site` so Blume can build full URLs:

```ts blume.config.ts
deployment: {
  site: "https://docs.example.com",
}
```

Without it, the image URL is emitted relative to the site.

## Meta tags

When an image is available, Blume adds the following to the page `<head>`,
alongside `og:title` and `og:description`:

```html
<meta
  property="og:image"
  content="https://docs.example.com/og/quickstart.png"
/>
<meta
  name="twitter:image"
  content="https://docs.example.com/og/quickstart.png"
/>
<meta name="twitter:card" content="summary_large_image" />
```

## Custom images

Override the generated card for any page with `seo.image` in frontmatter — point
it at a file in `public/` or an external URL:

```yaml
---
title: Pricing
seo:
  image: /og/pricing-custom.png
---
```

`seo.image` takes precedence over the generated card, and works even when
`og.enabled` is off — handy when you only want custom images on a few pages.

## Accent color

The card's mark and underline use your theme accent. Named accents (`blue`,
`green`, `orange`, `pink`, `purple`, `red`, `teal`) and raw hex values both work:

```ts blume.config.ts
theme: {
  accent: "teal",
}
```

:::note
OG rendering uses hex internally, so an `oklch` custom accent falls back to the
default. Give the accent a named preset or hex value if you use a custom palette.
:::
